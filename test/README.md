## 测试场景

### 向量搜索测试

- 语义相似性查询
- 跨领域文档检索
- 概念关联查询
  - "什么是导数？" 应该返回 calculus_fundamentals.md
  - "如何使用 Python 定义函数？" 应该返回 python_basics.md

### 全文搜索测试

- 精确关键词匹配
- 短语搜索
- 多词组合查询
  - "机器学习算法" 应该返回 machine_learning_intro.md
  - "薛定谔猫" 应该返回 quantum_physics_intro.md

### 基于标签的搜索测试

- 按类别筛选
- 标签组合查询
- 标签层级查询
  - "标签: 编程" 应该返回技术类文档
  - "标签: 历史 AND 标签: 战争" 应该返回 world_war_ii.md

### 混合搜索测试

- 向量搜索 + 全文搜索
- 标签过滤 + 向量搜索
- 多条件组合查询
  - "Python 中的数据类型" + 标签: 编程
  - "生态系统的能量流动" + 标签: 生物

## 使用建议

1. **测试向量搜索时**：使用语义查询，如"如何优化数据库性能？"
2. **测试全文搜索时**：使用精确关键词或短语，如"薛定谔方程"
3. **测试标签搜索时**：使用文档中的标签进行筛选和组合
4. **测试混合搜索时**：结合多种查询方式，如"标签:机器学习 AND 深度学习算法"
5. **评估检索质量**：检查返回结果的相关性、准确性和多样性

## 文档特点

- **多样性**：覆盖多个领域，测试系统的泛化能力
- **层次性**：每个领域包含不同主题，测试细分能力
- **关联性**：文档间存在概念关联，测试语义理解能力
- **完整性**：每个文档都有标签和关键词，支持多维搜索
- **可扩展性**：易于添加更多文档和标签

这些文档为全面测试 RAG 系统的检索能力提供了丰富的数据集。
